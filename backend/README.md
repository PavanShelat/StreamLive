# Backend — Supabase Schema & Edge Functions

## Directory Structure

```
backend/
└── supabase/
    ├── migrations/
    │   └── 001_initial_schema.sql   # Full DB schema
    └── functions/
        ├── livekit-token/           # Generates LiveKit JWT
        │   └── index.ts
        └── stream-metadata/         # Logs per-minute metadata
            └── index.ts
```

## Database Tables

| Table | Purpose |
|---|---|
| `profiles` | Extends Supabase auth.users with username/avatar |
| `streams` | Live and ended streams with viewer count |
| `follows` | Creator-follower relationships |
| `messages` | Chat messages with client_timestamp for offline ordering |
| `stream_metadata_log` | Per-minute viewer count + chat rate for highlight detection |
| `highlights` | Generated by n8n after stream ends |
| `notifications` | In-app notifications written by n8n |
| `daily_digests` | Daily top-streams digest generated by n8n cron |

## Edge Functions

### `livekit-token`
- **POST** `/functions/v1/livekit-token`
- **Auth:** Supabase JWT (Bearer token)
- **Body:** `{ "roomName": "stream-xxx", "role": "host" | "viewer" }`
- **Returns:** `{ "token": "<LiveKit JWT>", "identity": "<username>" }`

### `stream-metadata`
- **POST** `/functions/v1/stream-metadata`
- **Auth:** Supabase JWT (creator only)
- **Body:** `{ "streamId": "uuid", "viewerCount": 42 }`
- **Returns:** `{ "ok": true }`
- Called every 60s by the creator's app while streaming

## Deployment

```bash
npm install -g supabase

supabase login
supabase link --project-ref YOUR_PROJECT_ID

# Run the migration
supabase db push

# Set Edge Function secrets
supabase secrets set LIVEKIT_API_KEY=lk_api_xxx
supabase secrets set LIVEKIT_API_SECRET=your_secret
supabase secrets set SUPABASE_SERVICE_ROLE_KEY=your_service_role_key

# Deploy functions
supabase functions deploy livekit-token
supabase functions deploy stream-metadata
```
